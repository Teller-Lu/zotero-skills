# Zotero Full Translater（全文翻译式通俗解析）

在 `nature-reader` 的精神（忠实原文、绝不降级成摘要、源对齐）之上改造，产出**比精读报告更饱满、比逐句对照更通俗**的中文全文解析稿。
**核心目标（最高优先级）：把论文每一部分、每一段要表达的含义完整、清楚地讲出来——可以解析/转述，但不能把整段压成一句，不能曲解，不能省略关键信息；实在拿不准的，宁可直接翻译原文，也不要臆测。**

## 一、取数与读 PDF（同 zotero-read-explainer）

- **定位取数**：Zotero MCP（`zotero_search_items` → `zotero_get_item_metadata` / `zotero_get_item_children` / `zotero_get_annotations`）。附件路径取不到时，回落读 `prefs.js` 的 `extensions.zotero.dataDir`，PDF 在 `<dataDir>/storage/<附件key>/*.pdf`。
- **读 PDF**：**首选内置 Read 直接读 `.pdf`（靠 poppler / pdftoppm）**——一步到位、可 `pages` 选页、渲染清晰；仅当无 poppler / 需更高 DPI 抠密集公式 / 需裁某图区域时，才用 PyMuPDF 渲染成 PNG 兜底。关键数字用 `pdfplumber` 抽文本交叉核对。

## 二、输出规则（铁律）

详细产出契约与模板见 `references/output-spec.md`。七条铁律：

1. **正文主体（建模 / 方法 / 实验 / 讨论 / 结论）= 翻译 + 通俗解析，完整覆盖每一段**：用更通俗的话把每部分每段讲清楚；**不能曲解、不能省略关键信息**；**实在拿不准的段落，直接翻译原文**（宁可直译，不可臆测）。务必比"精读摘述"更饱满——读完应当等于读懂了正文细节。
2. **引言 / 相关工作 / 综述类 = 整体概述 + 参考文献现状表**：这类不逐句翻；先给一段整体概述，再用**表格**总结主要被引文献——每篇 1–2 句（做了什么 / 结论），并给出可查阅的主要信息（作者、年份、来源）。重点是"这些文献讲了啥、研究现状如何"。
3. **公式 = 全部保留，一条都不能删（最严格的要求）**：行内 `$...$`、独立 `$$...$$`，保留下标、分式、约束条件；看不清的公式也要占位列出并在文末标注不确定，**绝不删除任何一条公式**。
4. **图表 = 只放主要的**：仅给对理解最关键的主图 / 主表（重画或描述其含义即可），其余图表在正文用一句话提及，不全部搬运。
5. **结构 = 通篇标题序列**：用 `#` / `##` / `###` 对应原文章节层级，按原文顺序通读。
6. **两张附表**：① **符号 / 参数表**——列出全部符号、含义、**单位**；② **术语表**——中英对照。
7. **不确定清单**：把所有拿不准 / 存疑处统一列在文末（公式残缺、系数存疑、读图不确定等），并分清"影响理解"与"仅排版 / 系数级、不影响理解"。

## 三、与邻居 skill 的区别（避免误触发）

- `nature-reader`：逐句中英对照、近全文、不下判断 → 本 skill **更通俗、引言精简成文献表、只留主图表、且强制保留全部公式**。
- `zotero-read-explainer`：决策型**精读报告**（局限批判 + 工程价值，篇幅紧凑）→ 本 skill 是**全文饱满解析**（覆盖每段细节，不做决策批判）。
- `zotero-analytical-writer`：Obsidian **批量**笔记（公式靠 OCR）→ 本 skill **单篇**、公式靠渲染读图、产出独立 Markdown。

## 四、终检（回复前自检）

- **每一节、每一段的含义是否都讲清了？** 有没有整段被压成一句（不允许）？
- **公式是否一条都没删？** LaTeX 是否闭合（无未配对 `$`）？
- 引言是否给了"参考文献现状表"？图表是否只留了主要的？
- 符号 / 参数表（带单位）、术语表、文末不确定清单是否齐全？
- 标题序列是否覆盖了全文所有章节？
- 拿不准的地方是否改成了直接翻译、并登记进文末不确定清单？
