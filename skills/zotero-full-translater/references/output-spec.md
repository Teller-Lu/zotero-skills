# 全文翻译式通俗解析 —— 产出契约与模板

> 一份 Markdown 稿。落点由调用方指定（如某项目 `Reports/` 目录）。
> 总原则：**正文每部分每段含义完整讲清**（拿不准就直译）；**全部公式不删**；引言用文献表；只留主图表；带符号/参数表 + 术语表 + 文末不确定清单；通篇标题序列。

## 0. 元信息
- 标题（中 / 英）/ 作者 / 期刊·卷期·年 / DOI / 机构 / Zotero（item key、附件 key、标签）。
- 一句话定位。

## 1. 引言 / 研究背景（概述 + 文献现状表）
- 先一段整体概述（研究背景、痛点、本文切入点）。
- 再给**参考文献现状表**（只收主要被引文献）：

| 文献（作者, 年） | 来源 | 研究内容 / 结论（1–2 句） |
|---|---|---|
| … | … | … |

## 2…N. 正文各章（按原文标题序列，逐节翻译 + 通俗解析）
- 每个 `##` / `###` 对应原文一节；**逐段覆盖**，不漏段。
- 每段：用通俗中文把该段含义讲清（可解析、可重组语序），但**不省略关键信息、不曲解**；该段若含公式，**公式原样保留**（行内/独立 LaTeX）并解释各符号含义。
- **拿不准的段落直接翻译原文**，并在文末不确定清单登记。
- 主要图 / 表：在相应位置给出（重绘或文字描述其含义 + 数据要点）；非主要图表一句话带过。
