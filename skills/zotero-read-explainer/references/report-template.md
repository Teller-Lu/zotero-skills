# 精读报告模板（9 段式）

> 单篇论文决策型精读。三条铁律：①逻辑 / 方法 / 结论讲清；②**主要公式一条不少且要对**；③评价**平衡且决策导向**（有取舍前提、有价值判断、不空泛苛责）。
> 落点由调用方指定（如某项目的 `Reports/` 目录）。下列字段按需裁剪，但 0 / 3 / 5 / 7 / 8 段建议保留。

## 0. 元信息
- ID / 标题 / 一作＋合著 / 期刊＋卷期＋年 / 类型（综述·研究·会议）/ DOI / 机构
- Zotero：`item key`、附件 key、标签
- **一句话定位**：用一句话点出这篇到底做了什么、新在哪。

## 1. 问题与动机
- 研究对象与背景；行业 / 学术痛点；与已有工作的 gap（为什么还要做）。

## 2. 核心贡献
- 3–5 条，每条点出"**区别于谁、新在哪**"，避免复述摘要。

## 3. 方法与建模（核心，公式内联）
- 平台与对象；关键模型与公式（内联 LaTeX）；求解 / 控制策略。
- 对关键设计选择解释"**为什么这么做**"，而不仅是"做了什么"。

## 4. 仿真 / 实验验证
- 工况设定；对比基线；关键数值结果（**带方向与幅度**，如"相对基线 ↓13%"）。
- 注意把不同工况下的结论分开讲，别混为一谈。

## 5. 评价（平衡、决策导向）
- 先摆该领域**取舍优先级** → 据此判断作者取舍是否合理 → 区分"作者任务之外"与"我方落地必补"。
- 可设 `### 5.1` 深挖某个值得展开的方法论点（如某阈值如何科学确定）。

## 6. 对自身研究 / 工程的价值
- 可借鉴 / 可复用 / 可衍生（改进方向）/ 是否跟进（含等级判断与理由）。

## 7. 关键公式清单
- 逐条 LaTeX，编号尽量对应原文 Eq.，宁多勿漏。

## 8. 术语 / 记号（表格 ＋ 单位）

| 符号 | 含义 | 单位 |
|---|---|---|
| … | … | … |

## 9. 待原文核对
- **分级**：影响结论的存疑（需重点核）vs 仅排版 / 系数级（不影响理解）。分开写，不要把后者夸大成前者。
